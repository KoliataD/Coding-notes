Html Basic Coding Principles

Html Terminology

<>
this is an opening tag

</>
this is a closing tag

Elements
Things that are housed within these tags are elements, or properties. Elements will be the first thing that is written in a tag, it is also its main identifier.

Properties
You can give properties to your elements within tags this will modify things within that area of code.

Html Indentation and Spacing

When we are creating webpages we want to make sure that we use the same indendation, and spacing conventions throughout. Depending, on who you are working for, or what you are doing, you indentation may change. A great example is our spacing a lot of people use 4 spaces, some people use 2. I am currently using 2 because that is what the coding program epicodus has told me to do.

When we have a tag within a tag we indent the tag in the center. This rule sometimes does not apply to <!doctype> or <html> this is because if we indent everything that it will show up way to the left of our page. This is also why a lot of people like to use the double space instead of the quadruple space.

Whitespace
We want to avoid unnescassary whitespace. It is ok to leave spaces in code to seperate different functions, make code easier to read, and create divisions for easier access. Try not to use more than 1 whitespace.

Webpage Elements notes <!DOCTYPE html>
<html lang="eng-US">
</html>
This will go in all of your Html projects this will tell whatever program you are on what they are reading in this case it is html. The lang="eng-us" will make our webpage more accessible to people, by telling browsers what language our webpage its in we make it easier to translate.

<head>
</head>
<body>
</body>
The head element is what you will house all of your information, and links to your webpage files like if you are linking a css sheet, also for your title. The body is everything that will be showing up onto your page this will contain the sustenance of your webpage. Things that are contained in the head don't actually show up on the webpage it is more like the behind the scenes stuff that you want to effect your webpage like your css link sheet.

<title>title</title>
We can put this inside of our head tag this will change the title of our page to whatever is within it.

<div></div>
<span></span>
These are used to divide large sums of text we can also add classes to these to modify large areas of text. Div is what we will use when we are creating box models.

Box models
To make a box model we will create a class that we wil put inside of a div it will look like this <div class="box">. Than we will go in our style sheet and set a border color, type, and size, a padding, and a margin. That will create a box.

Linking Elements

Linking elements can link different files to a page.

<img>
This will create a image on your webpage. This tag requires a value for src, and optionally alt. Src will be the image adress, and alt will be alternative text.

it will look something like this:

<img src="img/image.jpeg" alt="this is a image">

<a>
This is a anchor element.This can add a link to text within it. This tag requires a value for href(hyperlinkreference). The value you will give href is the link to where you want it to go. When referencing within a page and not to an external link or internal file you will link it to an id with a # infront of it. When linking a image if you want it to be saved to your webpage you must save it to a folder called img than you must save the image with a name that ends in .jpeg .jpg. Than when referencing it you must write the entire path to navigate it to your img directory, and than to the picture itself.

it will look something like this:

<a href="whateveryourlinkis#this-goes-to-a-internal-id>this is the text it will be linked to</a>

Creating a project for Html

When creating a webpage we want a to make our main directory for the project which will just be the name of our project. Inside of our main projects directory we want to make a img file for any images we want to put in, a css file for any style sheets that we would like to add in, a readme with .md at the end so that it is read as markdown, our main webpage that will be the first one opened called index.html (this is automatically read as the main page/the homepage on github) and lastly all of our webpages that we want to make with the correct endings to make sure it is read in the code that we want it to be read in. After we are done with a project if it is going to be an actual webpage we want to get a MIT licence or some other form of licence for our website. If we are using github to publish our webpage we need to create a branch called gh-pages this is the branch that github will recognize as our website. If we decide to make our website on github the url will be my-username.github.io/repository-name. To update this we can just delete our gh-pages than create another branched named the same thing off of our main branch.
